# 概述

## 1.1 测试目的

本测试文档用于对“上海地铁路径规划与运营管理系统”进行系统化功能测试，验证系统各功能模块的正确性、稳定性、可靠性及性能表现。通过设计覆盖全面的测试用例，对系统的数据管理、路径规划、运营管理以及网络分析等功能进行验证，确保系统能够按照设计要求正确运行，并满足课程设计任务书规定的功能需求。

测试过程中须重点关注以下几个方面：

（1）验证地铁网络数据的正确导入与存储；

（2）验证最短时间路径、最少换乘路径等核心算法的正确性；

（3）验证站点关闭、线路停运、全网停运等运营管理功能的有效性；

（4）验证受影响区域分析、网络连通性分析等扩展功能的正确性；

（5）验证系统在大规模数据环境下的运行效率和稳定性；

（6）发现系统可能存在的逻辑错误、边界问题和异常情况，为系统优化和完善提供依据。

通过本测试文档的执行，可以全面评估系统的实现质量，为课程设计小组任务验收提供客观依据。

## 1.4 测试目标

测试的总体目标如下：

（1）验证系统功能实现的正确性；

（2）验证路径规划算法结果的准确性；

（3）验证运营管理功能对网络状态的影响是否符合设计要求；

（4）验证网络分析结果是否正确反映当前网络状态；

（5）验证系统在复杂场景和大规模数据环境下的稳定性；

（6）发现系统潜在缺陷并提出改进建议；

（7）确保系统满足课程设计验收要求。

测试完成后，应满足以下验收指标：

|      **项目**      | **指标** |
|:------------------:|:--------:|
|   功能测试通过率   |   ≥95%   |
|   核心功能通过率   |   100%   |
|    系统崩溃次数    |    0     |
|   数据读取成功率   |   100%   |
|   路径查询正确率   |   100%   |
| 网络状态更新正确率 |   100%   |
