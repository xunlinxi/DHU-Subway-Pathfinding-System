### 中等问题 (P1) - 影响功能完整性

#### P1-1: batchUpdateFromCSV 返回 bool 而非计数

**现象:** 函数返回 `bool`(成功打开文件), 测试期望 `int`(成功更新条数)
**影响:** 调用方无法知道实际更新了多少条

**修复建议:**
```cpp
// station.cpp:153
int StationManager::batchUpdateFromCSV(const std::string &csvPath) {
  // 返回成功更新的条数, -1 表示文件无法打开
  ...
  return success;
}
```

#### P1-2: 表头识别只匹配 'name' 英文

**现象:** `batchUpdateFromCSV` 跳过表头时仅判断 `fields[0] == "name"`，不匹配中文表头 `站点名称`
**影响:** 用户的 CSV 含中文表头时，第一行会被当作数据，导致 1 条记录被错误标记为失败

**修复建议:**
```cpp
// station.cpp:168
if (fields.size() >= 1 && (fields[0] == "name" || fields[0] == "站点名称"))
  continue;
```

#### P1-3: setStationStatus 不校验状态值

**现象:** 调用 `setStationStatus("X", "Y", "无效状态")` 不会返回 false, 站点 status 会被设为非法值
**影响:** 数据脏化，isClosed() 判断可能失效

**修复建议:**
```cpp
bool StationManager::setStationStatus(...) {
  if (newStatus != "开启" && newStatus != "关闭") return false;
  ...
}
```

#### P1-4: 空关闭列表提示不友好

**现象:** 当前无关闭站点时显示 `(无)`
**期望:** 显示 `所有站点均处于开放状态`

**位置:** [menu.cpp:215-228](file:///d:/Codes/%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84%E8%AF%BE%E7%A8%8B%E8%AE%BE%E8%AE%A1/menu.cpp#L215-L228) `Menu::showClosedStations()`

#### P1-5: 手工更新不支持多次连续操作

**现象:** 当前一次输入后即返回上级菜单
**期望:** 循环提示 '请输入待修改站点关键词（exit退出）'，支持连续多次更新

**位置:** [menu.cpp:192-211](file:///d:/Codes/%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84%E8%AF%BE%E7%A8%8B%E8%AE%BE%E8%AE%A1/menu.cpp#L192-L211) `Menu::toggleOneStation()`

#### P1-6: 起点/终点关闭时提示不明确

**期望:** `起点: XXX 已关闭,无法进行路径规划`
**当前:** 静默返回 invalid 路径

**修复建议:** 在 `Menu::runPathQuery` 中显式检查起终点状态

### 轻微问题 (P2) - 可选优化

#### P2-1: 4 号线内/外圈无标记

**现象:** 路径经过 4 号线时只显示 `4号线`，未区分内圈/外圈
