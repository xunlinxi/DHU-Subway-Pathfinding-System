# DHU-Subway-Pathfinding-System

DHU Data Structures Course Project: Shanghai Subway Pathfinding and Management System.
东华大学数据结构课程设计：上海地铁路径规划与管理系统。

---

## 1. 项目简介

基于 **现代 C++ (C++11)** 实现的上海地铁网络路径规划与管理系统，使用**邻接表**建模图结构，结合 **优先队列 + 状态扩展 Dijkstra + Yen K 短路算法**。

支持：

- 单条最优路径（时间最短 / 换乘最少）
- K 条候选路径（时间最短前 3 条 / 换乘最少前 3 条）
- 动态站点状态管理（开启 / 关闭 / 批量更新 / 一键恢复）
- 受影响区域分析（自动评估影响等级）
- 模糊搜索（部分汉字匹配 + 换乘站编号二次选择）
- **建站管理（可选加分项）**：添加 / 删除站点、手动加边
- **路径输出**：含「实际经过站数（不含换乘）」字段

## 6. 核心算法

| 类别 | 实现 |
|---|---|
| 图结构 | `vector<vector<Edge>>` 邻接表 |
| 最短路径 | 状态扩展 Dijkstra（`StateKey = (stationId, arrivedLine)`） |
| K 短路 | Yen 算法（偏离点 + 受限 Dijkstra + signature 去重） |
| 排序策略 | 双键优先队列（time+transfer）—— 时间最短相同时换乘最少优先 |
| 路径去重 | `signature()` 字符串哈希（节点序列 + 线路段序列） |
| 剪枝策略 | 关闭站点 / 过期状态 / blocked 节点 / 禁用边 |

### 6.1 算法亮点

- **状态扩展**：`(stationId)` → `(stationId, arrivedLine)`，正确处理换乘计数
- **双键代价**：`(主键, 副键)` 比较，time 优化时主键=time、副键=transfer
- **多重剪枝**：
  1. 关闭站点直接跳过
  2. 同 `(id, line)` 状态已存在且新代价不更优则不入队
  3. K 短路中 disabled 边 / blocked 节点 / 限定 root
- **受影响区域分析**：同线邻站 + 换乘线路数 + 孤立点检测 + 三级评估

## 7. 关键技术踩分点

- ✅ **邻接表 + 结构体**：`vector<vector<Edge>>`，`Station` / `Edge` / `Path` 全部为 struct
- ✅ **优先队列 + Dijkstra 变种**：`std::priority_queue` + 状态扩展
- ✅ **剪枝策略**：关闭站点 / 过期状态 / blocked 节点 / 禁用边
- ✅ **K 短路 + 路径去重**：Yen 算法 + signature 字符串哈希
- ✅ **双条件优先级**：时间最短相同时换乘最少优先
- ✅ **指定输出格式**：`A(L1) -> B(L1) -> C(L1) 换乘至 [L2] -> D(L2) -> E(L2)`
- ✅ **实际经过站数**：`actualStopCnt = stopCnt - transferCnt`
- ✅ **受影响区域分析**：同线邻站、孤岛检测、影响等级评估
- ✅ **模糊搜索**：部分汉字匹配 + 换乘站编号二次选择
- ✅ **批量更新 / 恢复初始**：CSV 双向同步
- ✅ **建站管理（加分项）**：添加 / 删除站点、手动加边
- ✅ **Unity Build**：`g++ main.cpp` 一行编译
- ✅ **--demo 演示模式**：跑预设测试用例，方便验收/截图
